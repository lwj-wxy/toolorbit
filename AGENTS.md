# Repository Agent Rules

本文件是仓库级强制规范，适用于所有协作代理/自动化工具（含 Codex、Cursor 及其他可读取仓库文档的工具）。

## 1. 调用前检查（每次必须执行）

1. 先确认需求范围：只允许改用户明确指定的页面、文件、功能。
2. 先读后改：必须先定位目标文件和代码位置。
3. 修改前检查工作区状态，识别并避开与本次任务无关的改动。
4. 若出现路径异常、权限异常、编码异常或不确定影响范围，立即停止并先反馈。

## 2. 文件与目录保护

1. 未经用户明确授权，禁止删除任何文件、目录、路由、导入导出、样式块。
2. 禁止把原目录内容迁移到新目录替代（例如 `src/views/xxx` 改到其他目录）。
3. 禁止顺手修改无关文件、批量替换无关内容、全局格式化无关文件。

## 3. 编码与中文规范

1. 所有文本文件必须使用 UTF-8（无 BOM）。
2. 写入后必须检查中文显示是否正常，禁止提交乱码。
3. 一旦发现乱码，先恢复编码问题，再继续业务修改。

## 4. 高风险操作约束

1. 涉及 `git checkout`、`git reset`、`git clean`、覆盖写入、批量替换前，必须先征得用户确认。
2. 任何可能导致历史丢失或文件不可逆变化的操作，默认禁止。

## 5. Vue 复杂度约束（computed / watch）

1. 页面级组件优先使用一个 `dataForm`（`reactive`）作为展示模型，避免把同一份数据拆成大量 `computed`。
2. `computed` 只用于复杂派生/多处复用场景；简单值映射在接口返回后一次性整理。
3. 默认每个页面组件 `watch` 不超过 1 个，仅用于路由参数或必要副作用；禁止使用多个 `watch` 做字段同步。
4. 超出上述约束时，必须在代码中写注释说明必要性。

## 6. 类型与初始值约束

1. `type` / `interface` 必须集中定义在文件顶部（`import` 之后、状态声明之前）。
2. 展示对象（如 `dataForm`）的字符串初始值统一使用空字符串 `""`，不允许以 `"—"` 作为初始化值。
3. `"—"` 仅可作为模板展示兜底文案，不可作为状态源数据。

## 7. 页面重置约束

1. 新路由页面默认不写 `resetXxxForm` / `resetDataForm`，避免冗余样板代码。
2. 仅在 `KeepAlive`、同页复用、局部状态需回滚时允许保留 reset 方法，并在代码中注释原因。
3. 常规详情页以 `detail = null` 和后续重新赋值驱动页面状态，不额外做无意义重置。

## 8. 无意义转换约束

1. 普通字段禁止套一层通用转换函数后再赋值，类型匹配时直接使用接口返回值。
2. 不得把每个字段都做统一“兜底转换”；仅特殊字段允许处理。
3. 特殊处理仅限：多字段拼接、日期/时间拆分、历史字段兼容、对象序列化展示。

## 9. 接口赋值约束

1. 详情数据默认通过 `Object.assign(target, response)` 回填。
2. 禁止对普通字段逐个手工映射构造。
3. 特殊字段仅做最小必要处理，不扩散到全字段。

## 10. 简单逻辑约束

1. 路由参数、基础空值判断、简单类型转换等场景，不允许额外抽无意义包装函数。
2. `route.query.id` 这类简单值应在使用处直接读取和判断，不再封装 `parseXxxId` 一类函数。
3. 只有在多处复用或逻辑明显复杂时，才允许抽成独立方法。

## 11. 方法定义约束

1. 页面业务方法统一使用箭头函数：`const fn = () => {}` / `const fn = async () => {}`。
2. 不再使用 `function fn() {}` 风格定义页面方法。
3. 仅类型定义或框架特殊要求场景允许例外。

## 12. 页面状态定义约束

1. 页面中的分页、列表、加载、统计等同类状态，默认统一收口到一个 `options = reactive({})` 对象中维护。
2. 简单页面不允许把 `page`、`pageSize`、`total`、`loading`、`list` 这类状态拆成多个独立 `ref`。
3. 仅当某个状态必须作为独立响应式引用复用时，才允许单独定义 `ref`，并且要有明确原因。

## 13. 变量命名约束

1. 页面中的局部变量、请求结果变量、查询参数变量必须使用有明确语义的命名，名称应能直接看出用途。
2. 不允许使用 `q`、`d`、`obj`、`item2`、`tmp` 这类无业务含义的缩写命名，除非是极小作用域且语义绝对明确的循环变量。
3. 查询参数优先命名为 `query`，接口结果数据按语义命名为 `detailData`、`listData`、`responseData`、`formData` 等可维护名称。
4. 命名优先保证维护可读性，不允许为了简写牺牲语义。

## 14. 详情页获取约束

1. 详情页默认使用 `const fetchDetail = async () => {}`。
2. 在 `fetchDetail` 内直接读取 `route.query.id` 并校验，不额外封装解析方法。
3. 请求成功后先落到中间变量，例如：`const { data: body } = await api(id); const data = body?.data;`。
4. `data` 为空时立即提示失败并返回，不继续后续处理。
5. 请求成功后默认 `Object.assign(dataForm, data)` 回填，特殊字段紧跟回填后处理。
6. 首次加载默认通过 `onMounted(async () => { await fetchDetail(); })` 触发。

## 15. 交付前核对

1. 仅包含目标需求相关变更。
2. 无误删文件、无新增异常目录。
3. 无乱码、无残留无用逻辑。
4. 明确列出变更文件与原因。
